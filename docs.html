Спецификация интеграции Core API

Настоящий документ содержит исчерпывающее техническое описание интерфейсов взаимодействия распределенной сети ShadowsLTD Core. Платформа спроектирована для обеспечения атомарных транзакций обмена метаданными между изолированными транспортными узлами (Edge Nodes) в условиях асимметричных сетевых задержек.

1. Протокол криптографической верификации сессий

Все входящие вызовы к магистральным шлюзам верхнего уровня (Upstream Gateways) требуют обязательной пре-авторизации пакетов в рамках нулевого транспортного слоя (L0). Сессионное рукопожатие осуществляется с использованием гибридного шифрования на базе эфемерных ключей Диффи-Хеллмана на эллиптических кривых (ECDHE-X25519) со сквозным контролем целостности HMAC-SHA256.

2. Параметры авторизационных заголовков

Передача служебных метаданных должна строго соответствовать RFC-инструкциям кластера. Использование кастомных прокси-заголовков приводит к немедленной инвалидации сессионного конвейера балансировщика трафика.

Заголовок Тип Описание назначения и область применения
X-Core-Node-ID String (Hex) Уникальный криптографический идентификатор верифицированного узла кластера ShadowsLTD.
X-Routing-Signature String (Base64) Подпись тела запроса, вычисленная с использованием приватного ключа сессии.
X-Cluster-Epoch Integer Контрольная временная метка для предотвращения векторов атак повторения (Replay).

3. Алгоритмы троттлинга и оркестрации очередей

Для предотвращения каскадных отказов (Cascading Failures) и деградации пропускной способности распределенных хранилищ, на пограничных маршрутизаторах развернута адаптивная модель Token Bucket. Лимиты распределяются динамически, основываясь на текущем весовом коэффициенте репутации IP-адреса источника и индекса утилизации ресурсов центрального процессора вызываемого вычислительного кластера ShadowsLTD.
